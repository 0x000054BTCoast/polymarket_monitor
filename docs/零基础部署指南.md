# polymarket-monitor 零基础部署指南（2026 最新代码版）

> 目标读者：**0 编程经验**用户。  
> 目标：在你自己的电脑上把项目跑起来，并能在浏览器看到页面与接口返回。  
> 适用代码：本仓库当前 `main`/当前分支最新代码（FastAPI + SQLite + Next.js）。

---

## 你部署完成后会看到什么

启动成功后，你可以访问：

- 前端页面：<http://localhost:3000>
- 后端健康检查：<http://localhost:8000/health>

`/health` 预期返回：

```json
{"status":"ok"}
```

如果这两个地址都正常，说明部署成功。

## 一、先安装 3 个基础工具（只做一次）

请先安装：

1. **Git**（下载代码）
2. **Python 3.11+（建议 3.11.x）**（后端运行）
3. **Node.js 18+（建议 20 LTS）**（前端运行）

### 1）检查工具是否可用

打开终端（Windows 可用 PowerShell，macOS 用 Terminal），依次执行：

```bash
git --version
python3 --version
# 如果系统没有 python3，再试：python --version
node --version
npm --version
```

每条命令都能输出版本号即可继续。

---

## 二、下载项目代码

在终端执行（把 `<你的仓库地址>` 改成真实地址）：

```bash
git clone <你的仓库地址> polymarket-monitor
cd polymarket-monitor
```

---

## 三、创建环境变量文件（推荐默认不改）

项目根目录有模板 `.env.example`，先复制为 `.env`：

```bash
cp .env.example .env
```

Windows PowerShell：

```powershell
Copy-Item .env.example .env
```

### 可以先不改 `.env`

默认配置已可本地运行：

- 后端读取 `http://localhost:8000`
- 前端默认请求 `NEXT_PUBLIC_API_BASE=http://localhost:8000`

如果你后面改了后端端口，再同步修改 `.env` 中的 `NEXT_PUBLIC_API_BASE`。

---

## 四、启动后端（FastAPI）

### 方案 A（推荐，适合 macOS / Linux）

在项目根目录执行：

```bash
make backend-install
make backend-run
```

看到类似以下内容即成功：

- `Uvicorn running on http://127.0.0.1:8000`

> 第一次安装依赖会稍慢（需要下载 Python 包），属正常现象。

### 方案 B（没有 make，或 Windows 用户推荐）

> `make backend-install` 中使用的是类 Unix 的虚拟环境激活方式。Windows 原生环境建议按下面手动步骤执行。

#### macOS / Linux 手动命令

```bash
python3 -m venv .venv
source .venv/bin/activate
pip3 install -r backend/requirements.txt
cd backend
PYTHONPATH=. uvicorn app.main:app --reload --port 8000
```

#### Windows PowerShell 手动命令

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r backend\requirements.txt
cd backend
$env:PYTHONPATH='.'
uvicorn app.main:app --reload --port 8000
```

## 五、启动前端（Next.js）

> 需要**新开一个终端窗口**，后端窗口保持运行。

在项目根目录执行：

```bash
make frontend-install
make frontend-run
```

看到类似 `Local: http://localhost:3000` 即表示前端启动成功。

### 为什么建议用 `make frontend-install`

这个命令会自动清理常见代理变量，并固定 npm 官方源，能减少如下报错：

- `ECONNREFUSED 127.0.0.1:<port>`
- 因本机/公司代理配置残留导致的 npm 安装失败

## 六、验收（按顺序检查）

1. 打开前端：<http://localhost:3000>
2. 打开健康检查：<http://localhost:8000/health>
3. （可选）打开系统状态：<http://localhost:8000/api/system/status>

验收标准：

- 页面能打开且有内容
- `/health` 返回 `{"status":"ok"}`
- 系统状态接口能返回 JSON

## 七、常见问题（小白高频）

### 1）`make: command not found`

你没有安装 `make`。直接按上文“手动命令”运行即可。

### 2）`python` / `python3` 找不到

说明 Python 未安装或未加入 PATH。请重装 Python 并勾选 “Add Python to PATH”。

- macOS / Linux 常用 `python3`、`pip3`
- Windows 常用 `python`、`pip`

### 3）`npm` 不是内部或外部命令

说明 Node.js 未安装，安装后重试。

### 4）3000 或 8000 端口被占用

- 关闭占用端口的程序（常见是另一个前端/后端项目）
- 或改端口运行（改后端端口时要同步改 `.env` 的 `NEXT_PUBLIC_API_BASE`）

### 5）前端打开了，但数据为空/报错

依次检查：

1. 后端窗口是否仍在运行
2. <http://localhost:8000/health> 是否正常
3. `.env` 中 `NEXT_PUBLIC_API_BASE` 是否和后端实际地址一致

### 6）想先“只看界面”，不依赖后端

当前前端支持后端不可用时的降级/Mock 展示模式（界面会标记 `MOCK/DEMO`），可用于先验证页面可打开。

## 八、第二次以后怎么启动（最常用）

每次开发/查看时，只需开两个终端：

- 终端 1（后端）

  ```bash
  cd polymarket-monitor
  make backend-run
  ```

- 终端 2（前端）

  ```bash
  cd polymarket-monitor
  make frontend-run
  ```

> 如果你重装过环境、删除过 `.venv` 或 `node_modules`，再执行一次安装命令：
> `make backend-install` / `make frontend-install`。

---

## 九、如何停止服务

在各自终端按：

- Windows / Linux：`Ctrl + C`
- macOS：`Control + C`

即可停止。
